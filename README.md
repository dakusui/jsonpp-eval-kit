# JSON++ / jq++ Evaluation Kit

Thank you for helping evaluate **JSON++ / jq++**, a lightweight way to reduce
duplication in JSON/YAML configuration. This is a small hands-on task (about
60–90 minutes); your experience will inform an IEEE Software article, and you will
be acknowledged in it (with your consent).

## Start here

1. Read the task brief — **[`task-brief.md`](./task-brief.md)** (日本語:
   **[`task-brief.ja.md`](./task-brief.ja.md)**).
2. Set up the tools with **[`getting-started.md`](./getting-started.md)** (日本語:
   **[`getting-started.ja.md`](./getting-started.ja.md)**).
3. Do the task on the workflows in **[`starter-corpus/`](./starter-corpus/)**
   (optionally also your own, in [`your-corpus/`](./your-corpus/)).
4. Fill in **[`questionnaire.md`](./questionnaire.md)** (日本語:
   **[`questionnaire.ja.md`](./questionnaire.ja.md)**).

## Submitting

Email your working set (see the task brief) and the completed questionnaire to
**dakusui [at] gmail [dot] com**.

Please **email any questions privately** and **do not open public issues or pull
requests** — this keeps the exercise fair for the other participants.

Your responses are used only for this evaluation and the resulting IEEE Software
article, per the consent options in the questionnaire.

# JSON++ / jq++ 評価キット

**JSON++ / jq++**（JSON/YAML 設定の重複を減らす軽量な手法）の評価にご協力いただき、
ありがとうございます。所要時間は 60〜90 分程度の小さな実践課題です。いただいた経験は
IEEE Software 誌の論文に反映され、ご同意のうえで謝辞に記載させていただきます。

## はじめに

1. 課題のご案内 **[`task-brief.ja.md`](./task-brief.ja.md)** を読んでください。
2. **[`getting-started.ja.md`](./getting-started.ja.md)** でツールを準備します。
3. **[`starter-corpus/`](./starter-corpus/)** のワークフローに対して課題を行います
   （任意で、ご自身のものを [`your-corpus/`](./your-corpus/) に置いても構いません）。
4. **[`questionnaire.ja.md`](./questionnaire.ja.md)** にご記入ください。

## ご提出

作業一式（課題のご案内を参照）と記入済みのアンケートを
**dakusui [at] gmail [dot] com** までメールでお送りください。

ご質問は**個別にメールで**お願いします。また、他の参加者のために
**公開の Issue や Pull Request は作成しないで**ください。

いただいた回答は、この評価および結果として執筆される IEEE Software 誌の論文の
ためにのみ使用します（アンケート内の同意選択に従います）。
